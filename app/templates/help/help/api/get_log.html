Get information about a log

Returns information about a given log. Takes activity_log_id (log primary key, as an integer) as an argument.

Endpoint /_api/get/log/[ACTIVITY_LOG_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON object
  • { 'id': Primary key (int), 'article_id': Primary key for article that was viewed (int), 'anonymous_uuid': Tracking UUID for anonymous users (str), 'user_id': User primary key for logged in users (int), 'share_id': Share primary key (if applicable) (int), 'ip_address': IP address for the request (str), 'referrer_page': Where did the user come from? (str), 'user_agent': Browser and operating system information (str), 'timestamp': UTC timestamp (datetime) }
Example response { 'id': 7309, 'article_id': 63, 'anonymous_uuid': 'd2065a7d-4352-46a1-945c-74cd4bc7a243', 'user_id': null, 'share_id': 649, 'ip_address': '156.49.238.72', 'referrer_page': 'https://www.facebook.com/zuck', 'user_agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/88.0.4324.190 Safari/537.36', 'timestamp': 2021-02-26T00:39:44.126285Z }
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: Log not found.
  • 429 (Too many requests): You have exceeded your rate limit.
