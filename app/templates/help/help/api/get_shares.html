Get a list of shares

Returns a list of shares for a given article. Takes article_id (article primary key, as an integer) as an argument.

Endpoint /_api/get/shares/[ARTICLE_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON array of objects
  • [ { 'id': Primary key (int), 'share_uuid': Share UUID (str), 'article_id': Article primary key (int), 'timestamp': UTC timestamp (datetime) }, { ... } ]
Example response [{ 'id': 1, 'share_uuid': '0c297f75-3800-4ed3-b39f-df4c97ec8a4b', 'article_id': 5, 'timestamp': 2021-02-26T00:56:12.1263485Z }, { 'id': 2, 'share_uuid': '9f8eeef7-89aa-4705-ace7-0f489d9e3d3f', 'article_id': 5, 'timestamp': 2021-02-26T04:56:12.143485Z }]
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: Article not found.
  • 429 (Too many requests): You have exceeded your rate limit.
