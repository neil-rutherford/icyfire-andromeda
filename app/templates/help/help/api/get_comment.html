Get information about a comment

Returns information about a given comment. Takes comment_id (comment primary key, as an integer) as an argument.

Endpoint /_api/get/comment/[COMMENT_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON object
  • { 'id': Primary key (int), 'article_id': Article primary key (int), 'user_id': Commenter's user primary key (int), 'comment': Comment body (str), 'timestamp': UTC timestamp (datetime) }
Example response { 'id': 5235, 'article_id': 3, 'user_id': 32, 'comment': "idk wat ur talking about sweaty :)", 'timestamp': 2021-02-26T00:39:44.126285Z }
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: Comment not found.
  • 429 (Too many requests): You have exceeded your rate limit.
