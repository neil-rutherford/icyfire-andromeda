Create a user account

Creates a new user account.

Endpoint /_api/post/users
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET], 'email': Contact email address (str, max 254 characters), 'password': Password (str, recommended 8 or more characters), 'username': Username (str, max 100 characters, must be unique), 'yob': Year of birth (int), 'gender': Gender (int, 1: Male, 2: Female) }
Example POST input { 'consumer_key': "c0n5um3r_k3y", 'consumer_secret': "c0n5um3r_53cr37", 'email': 'herbert_garrison@southparkelementary.edu', 'password': '1@l0v3#mr$h47%', 'username': 'real_mr_garrison', 'yob': 1980, 'gender': 1 }
On success
  • 201 status code
  • JSON object
  • { 'status': 201, 'response': "SUCCESS: User [USERNAME] created. }
Error messages
  • 400: Some sort of error occurred when creating the article object.
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 429 (Too many requests): You have exceeded your rate limit.
