Get information about a relationship between two users

Returns information about a relationship between two users. Takes user1_id (user 1 primary key, as an integer) and user2_id (user 2 primary key, as an integer) as arguments.

Endpoint /_api/get/connection/[USER1_ID]&[USER2_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON object
  • { 'user1_follows_user2': Does User 1 follow User 2? (bool), 'user2_follows_user1': Does User 2 follow User 1? (bool), 'user1_messages_user2': How many messages has User 1 sent to User 2? (int), 'user2_messages_user1': How many messages has User 2 sent to User 1? (int) }
Example response { 'user1_follows_user2': true, 'user2_follows_user1': false, 'user1_messages_user2': 413, 'user2_follows_user1': 3 }
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: Can't find one of the users.
  • 429 (Too many requests): You have exceeded your rate limit.
