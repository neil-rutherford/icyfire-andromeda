Get a list of articles

Returns a list of the 25 most recent articles.

Endpoint /_api/get/articles
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON array of objects
  • [ { 'id': Primary key (int), 'title': Article title (str), 'description': Article description (str), 'timestamp': UTC timestamp (datetime), 'comments': Number of comments (int), 'logs': Number of page views (int), 'shares': Number of social media shares (int) }, { ... } ]
Example response [{ 'id': 4, 'title': 'U.S. strikes Iran-backed militia facilities in Syria', 'description': 'The United States on Thursday carried out an airstrike against facilities in Syria linked to an Iran-backed militia group, the Pentagon announced.', 'timestamp': 2021-02-26T00:39:44.126285Z, 'comments': 56, 'logs': 4521, 'shares': 32 }, { 'id': 5, 'title': 'Senate parliamentarian rules $15 minimum wage cannot be included in relief package', 'description': 'The Senate parliamentarian ruled Thursday that the provision to increase the minimum wage to $15/hour cannot be included in the broader $1.9 trillion COVID relief package.', 'timestamp': 2021-02-26T00:56:12.1263485Z, 'comments': 12, 'logs': 523, 'shares': 43 }]
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 429 (Too many requests): You have exceeded your rate limit.
