Get information about a user

Returns information about a given user. Takes user_id (user primary key, as an integer) as an argument.

Endpoint /_api/get/user/[USER_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON object
  • { 'id': Primary key (int), 'email': User's email address (str), 'expires_on': When the user's Patriot subscription expires in UTC (datetime), 'username': User's username (str), 'status': User's status (str), 'dob': User's date of birth (datetime), 'gender': User's gender (int, 1: Male, 2: Female), 'email_opt_in': Has the user opted in to receiving news roundups in their inbox? (bool), 'is_verified': Has the user verified their email address? (bool), 'profile_pic': URL for the user's profile picture (str), 'coins': How many coins does the user have? (int) }
Example response { 'id': 4, 'email': 'e.cartman@southparkelementary.edu', 'expires_on': 2021-05-11T00:44:55.126285Z, 'username': 'real_eric_cartman', 'status': 'Screw you guys, I'm going home!', 'dob': 2011-01-01, 'gender': 1, 'email_opt_in': false, 'is_verified': true, 'profile_pic': 'https://bit.ly/2NzXfsh', 'coins': 58 }
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: User not found.
  • 429 (Too many requests): You have exceeded your rate limit.
