Create an app

Creates a Galatea API app.

Endpoint /_api/post/clients
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET], 'user_id': User ID to be linked to the app (int), 'is_articles': Applying for articles permission? (bool), 'is_comments': Applying for comments permission? (bool), 'is_logs': Applying for logs permission? (bool), 'is_shares': Applying for shares permission? (bool), 'is_users': Applying for users permission? (bool), 'is_connections': Applying for connections permission? (bool), 'is_messages': Applying for messages permission? (bool), 'is_api': Applying for API permission? (bool), 'company_name': Company name if applicable, else "Individual" (str, max 300 characters), 'contact_name': Point of contact for this application (Surname, Given) (str, max 300 characters), 'phone_number': +1 country code phone number (str, max 14 characters), 'email': Contact email for this application (str, max 254 characters), 'use_case': How do you plan to use the Galatea API? (str), 'developer_platform': What platform are you developing this app on? (str, max 100 characters), 'monetization_model': How do you plan to monetize this app, if applicable, else "N/A" (str), 'permission_justification': Explain why your app needs the permissions you requested (str) }
Example POST input { 'consumer_key': "c0n5um3r_k3y", 'consumer_secret': "c0n5um3r_53cr37", 'user_id': 42, 'is_articles': false, 'is_comments': true, 'is_logs': false, 'is_shares': false, 'is_users': false, 'is_connections': false, 'is_messages': true, 'is_api': false, 'company_name': "Individual", 'contact_name': 'Dixon-Coffey, Duncan', 'phone_number': '(123) 456-7890', 'email': 'dixcof@gmail.com', 'use_case': 'I want to send someone a message if they comment on one of my articles.', 'developer_platform': 'Python script running on AWS', 'monetization_model': 'N/A', 'permission_justification': 'Comments permission to get author of the comment\nMessages permission to send them a message.' }
On success
  • 201 status code
  • JSON object
  • { 'status': 201, 'response': "SUCCESS: API client application [CLIENT_ID] created." }
Error messages
  • 400: Some sort of error occurred when creating the client object.
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 429 (Too many requests): You have exceeded your rate limit.
