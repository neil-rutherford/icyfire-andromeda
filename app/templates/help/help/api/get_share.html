Get information about a share

Returns information about a given share. Takes share_id (share primary key, as an integer) as an argument.

Endpoint /_api/get/share/[SHARE_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 200 status code
  • JSON object
  • { 'id': Primary key (int), 'share_uuid': Share UUID (str), 'article_id': Article primary key (int), 'anonymous_uuid': Tracking UUID for anonymous users (str), 'user_id': User primary key for logged in users (int), 'ip_address': IP address for the request (str), 'network': Network code (1: Facebook, 2: Twitter, 3: Tumblr, 4: LinkedIn) (int), 'timestamp': UTC timestamp (datetime) }
Example response { 'id': 43, 'share_uuid': '0c297f75-3800-4ed3-b39f-df4c97ec8a4b', 'article_id': 542, 'anonymous_uuid': null, 'user_id': 6329, 'ip_address': '206.232.249.54', 'network': 1, 'timestamp': 2021-02-26T00:39:44.126285Z }
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: Share not found.
  • 429 (Too many requests): You have exceeded your rate limit.
