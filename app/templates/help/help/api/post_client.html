Modify an app's permissions

Modifies the status of an app. Takes api_client_id (API client primary key, as an integer) as an argument.

Endpoint /_api/post/client/[API_CLIENT_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET], 'is_approved': If true, the app can begin to make requests (bool), 'is_blacklist': If true, the app can no longer make requests (bool), 'is_articles': Articles permission (bool), 'is_comments': Comments permission (bool), 'is_logs': Logs permission (bool), 'is_shares': Shares permission (bool), 'is_users': Users permission (bool), 'is_connections': Connections permission (bool), 'is_messages': Messages permission (bool), 'is_api': API permission (bool), }
Example POST input { 'consumer_key': "c0n5um3r_k3y", 'consumer_secret': "c0n5um3r_53cr37", 'is_approved': true, 'is_blacklist': false, 'is_articles': true, 'is_comments': true, 'is_logs': false, 'is_shares': false, 'is_users': true, 'is_connections': true, 'is_messages': false, 'is_api': false }
On success
  • 200 status code
  • JSON object
  • { 'status': 200, 'response': 'SUCCESS: API Client [CLIENT_ID] updated.' }
Error messages
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 404: Client not found.
  • 429 (Too many requests): You have exceeded your rate limit.
