Send a message

Sends a message.

Endpoint /_api/post/messages
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET], 'reply_to': Message primary key (int) if this is a reply, else None, 'to_user_id': Recipient user ID (int), 'subject': Subject line (str, 50 character max), 'body': Message body (str, 300 character max) }
Example POST input { 'consumer_key': "c0n5um3r_k3y", 'consumer_secret': "c0n5um3r_53cr37", 'reply_to': null, 'to_user_id': 420, 'subject': 'Lunch at Olive Garden?', 'body': "Hey,\n Wanna grab lunch today at the Olive Garden down the street?" }
On success
  • 201 status code
  • JSON object
  • { 'status': 201, 'response': "SUCCESS: Message sent to [TO_USER_ID]." }
Error messages
  • 400: Some sort of error occurred when creating the message object.
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. (e.g. Trying to access users without the "users" permission.)
  • 429 (Too many requests): You have exceeded your rate limit.
