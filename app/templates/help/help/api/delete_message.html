Delete a message

Deletes a message. Requires the linked user account to be the sender or recipient of the message. Takes message_id (message primary key, as an integer) as an argument.

Endpoint /_api/delete/message/[MESSAGE_ID]
Method POST
Form-encoded POST input { 'consumer_key': [CONSUMER_KEY], 'consumer_secret': [CONSUMER_SECRET] }
On success
  • 204 status code
  • JSON object
  • { 'status': 204, 'response': "SUCCESS: Message [MESSAGE_ID] deleted. }
Error messages
  • 400: Some sort of error occurred when deleting the article object.
  • 403 (Client not found): Consumer key is incorrect.
  • 403 (Incorrect credentials): Consumer secret is incorrect.
  • 403 (API access has been restricted): Your credentials are correct, but your app is either not approved or on a blacklist.
  • 403 (Insufficient permissions): You don't have permission to access that resource. Could also be trying to delete a message that isn't yours.
  • 404: Article not found.
  • 429 (Too many requests): You have exceeded your rate limit.
